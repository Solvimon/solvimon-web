# Translations

This project uses [FormatJS](https://formatjs.io/) for internationalisation. All translatable strings are extracted from source code into a central catalog, and each supported locale has its own JSON file generated from that catalog.

## File structure

```
src/translations/
  source.json      — auto-generated message catalog (source of truth, do not edit manually)
  supported.json   — list of locale codes that must be fully translated
  en-US.json       — compiled English translations (generated from source.json)
  nl-NL.json       — Dutch translations
```

`source.json` is regenerated by the extract script and must not be edited by hand. All other locale files are derived from it.

## Scripts

| Script | What it does |
|---|---|
| `npm run translations:extract` | Re-scans `src/**/*.{ts,vue}` and regenerates `source.json` |
| `npm run translations:compile` | Compiles `source.json` → `en-US.json` (run after extract) |
| `npm run translations:check` | Verifies every key in `source.json` exists in all locale files |

## Adding a translatable string

Use the `$t` function (or `useI18n().t`) with a string literal ID and a `defaultMessage`:

```ts
const label = $t('my_feature.submit_button', { defaultMessage: 'Submit' })
```

The ID must be a string literal — dynamic IDs are not extracted. After adding new strings, regenerate the catalog:

```sh
npm run translations:extract
npm run translations:compile   # keeps en-US.json in sync
```

## Updating translations

To fill in missing translations for all locales, run the `/translate` Claude Code slash command from the project root. It extracts, identifies missing keys, generates translations for each locale, and verifies completeness.

```sh
/translate           # all locales
/translate nl-NL     # one locale only
```

To translate a specific locale manually, edit `src/translations/<locale>.json` directly. Keys must match `source.json` exactly. Strings use [ICU MessageFormat](https://unicode-org.github.io/icu/userguide/format_parse/messages/) — preserve any `{variable}`, plural, or select blocks and translate only the surrounding text.

## Adding a new locale

1. Add the locale code to `src/translations/supported.json`.
2. Run `/translate <locale>` to generate the initial translations.
3. Verify with `npm run translations:check`.

## CI check

The pipeline runs `npm run translations:check` on every push. It exits non-zero if any key in `source.json` is absent from any locale file listed in `supported.json`. Resolve failures by running `/translate` and committing the updated locale files.
